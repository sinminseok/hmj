## Home And Job 프로젝트

-----

### 프로젝트 설명
`사용자들이 호주에서 집과 일자리를 쉽게 구할 수 있게 도움을 주는 서비스`

### 프로젝트 구조
계층형 구조이면 프로젝트가 커질 수록 클래스들이 응집될 수 있기에 `도메인형 구조` 선택

### 도메인 설계 원칙

- 무분별한 Setter은 사용하지 말것
- 도메인을 단일 책임 원칙에 따라 나눠 설계할 것
- DDD 설계

### Service 역할
- Entity <-> DTO 변환
- 

### Repository 역할
- DB 연결, 접근

### Controller 역할
- API 요청 송,수신
- 도메인을 모르게 설계(DTO 사용)
